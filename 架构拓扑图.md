## 图 1 — 系统总拓扑（端到端）

从用户、编排、LangGraph、大模型形态到 MCP 与外部依赖的单张总览。

```mermaid
flowchart TB
  subgraph External["外部依赖"]
    DS[(DashScope / OpenAI 兼容 LLM)]
    BS[(Baostock)]
    NW[(互联网: 新闻爬取等)]
  end

  subgraph UI["表现层"]
    APP[Streamlit app.py]
  end

  subgraph ORCH["编排层 main.py + orchestration"]
    META[extract_metadata]
    VAL[validate_ticker]
    CACHE[(analysis_cache)]
    PRE[build_prefetch_context]
    ST[AgentState]
    STR[stream_analysis / astream]
    LOG[runtime_logger]
  end

  subgraph LG["LangGraph"]
    START((START))
    F[fundamental]
    T[technical]
    V[valuation]
    N[news]
    S[summary]
    ENDN((END))
    START --> F & T & V & N
    F & T & V & N --> S --> ENDN
  end

  subgraph AG["智能体 agents"]
    direction LR
    R4[四维: ReAct + user 消息]
    RS[总结: system + user 无工具]
  end

  subgraph MCP["MCP"]
    MCL[mcp_client 独立线程]
    MSV[tools.mcp_server 子进程]
    T28[28 工具 → Baostock / 爬虫 / ...]
    MCL <-->|stdio| MSV
    MSV --> T28
  end

  APP --> META
  META --> DS
  META -.-> CACHE
  VAL --> BS
  VAL -.-> CACHE
  PRE --> MCL
  PRE --> ST
  META --> VAL
  VAL -->|通过| ST
  ST --> STR
  STR --> LG
  STR --> LOG
  F & T & V & N --> R4
  S --> RS
  R4 <--> MCL
  R4 --> DS
  RS --> DS
  T28 --> BS
  T28 --> NW
```

## 图 3 — MCP 客户端与进程模型

说明为何使用**独立线程 + 专用事件循环**承载 MCP（避免 LangGraph 并行节点与 stdio MCP 的 anyio cancel scope 冲突）。

```mermaid
flowchart TB
  subgraph Main["主线程: LangGraph / asyncio"]
    G[astream 各节点]
    A1[Agent ReAct]
    G --> A1
  end

  subgraph MCPThread["守护线程: MCP 专用事件循环"]
    LOOP[event loop]
    SESS[ClientSession]
    LOOP --- SESS
  end

  subgraph Sub["子进程"]
    STD[stdio MCP Server]
    REG[TOOLS 注册表 28 tools]
    STD --> REG
  end

  A1 -->|call_mcp_tool_async / StructuredTool| SESS
  SESS <-->|stdin/stdout| STD
```
